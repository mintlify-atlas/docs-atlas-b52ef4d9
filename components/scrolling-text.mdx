`ScrollingText` measures its container and content on mount and on resize. If the text overflows, it duplicates the content and applies a CSS keyframe animation to create a seamless looping marquee. When text fits, it simply truncates with an ellipsis. It supports plain text, external links, and arbitrary Svelte snippet children.

## Props

<ParamField path="text" type="string" default="''">
  Plain text string to display. Mutually exclusive with `children` â€” if `children` is provided, `text` is ignored for rendering (though it still triggers the overflow recalculation effect).
</ParamField>

<ParamField path="href" type="string | undefined" default="undefined">
  When set alongside `text`, wraps the text in an `<a>` tag with `target="_blank"` and `rel="noopener noreferrer"`. The link gets an animated underline that slides in on hover. Has no effect when `children` is provided.
</ParamField>

<ParamField path="className" type="string" default="''">
  Additional Tailwind or CSS classes applied to the outer container `div`. Use this to control font size, color, and other typography styles from the parent.
</ParamField>

<ParamField path="fadeWidth" type="number" default="8">
  Width of the left/right fade mask in percentage points. When text overflows, a `mask-image` gradient fades the edges. Increase this value for a wider fade, decrease for a tighter one.
</ParamField>

<ParamField path="children" type="Snippet | undefined" default="undefined">
  A Svelte snippet rendered instead of `text`/`href`. Use this when you need formatted content inside the scroller â€” for example, multiple `<a>` tags for artist names in `StatsFM`. The duplicate copy (rendered `aria-hidden`) is also rendered from the same snippet.
</ParamField>

## How it works

### Overflow detection

On mount and on every `resize` event, `calculateOverflow` compares `textRef.scrollWidth` with `containerRef.offsetWidth`. If the text is wider than the container, `isOverflowing` is set to `true`.

Once in overflow mode, the component uses the distance between `textRef` and `dupeTextRef` (the duplicate node) as the scroll length â€” this accounts for the `ãƒ»` divider between the two copies.

### Scroll speed

Animation duration is calculated from the pixel length of the content divided by a fixed speed of 30 pixels per second:

```js
const pixelsPerSecond = 30;
animationDuration = len / pixelsPerSecond;
```

This keeps the apparent speed constant regardless of text length â€” shorter text scrolls more quickly (shorter duration), longer text scrolls at the same perceived pace.

### Edge fade

When overflowing, a CSS `mask-image` gradient fades both edges. The `leftStop` and `rightStop` positions are calculated as percentages of the container width, offset by `fadeWidth`:

```js
let leftStop  = $derived(`${(8 / containerWidth) * 100 + fadeWidth / 2}%`);
let rightStop = $derived(`calc(100% - ${leftStop})`);
```

### Duplicate copy and divider

A `ãƒ»` character is inserted between the original and duplicate as a visual separator. The duplicate carries `aria-hidden="true"` so screen readers see only one copy of the text. The duplicate `<a>` also gets `tabindex="-1"` to keep it out of the focus order.

## Usage

```svelte
<script>
  import ScrollingText from '$lib/components/ui/ScrollingText.svelte';
</script>

<!-- Plain text, truncates if it fits -->
<ScrollingText
  text="A very long track name that might overflow its container"
  className="text-lg font-bold text-white"
/>

<!-- Text with an external link -->
<ScrollingText
  text="Aphex Twin"
  href="https://open.spotify.com/artist/6kBDZFXuLrZgHnvmPu9NsG"
  className="text-sm text-gray-400"
/>

<!-- Custom snippet children (e.g. multiple linked artists) -->
<ScrollingText className="text-sm text-gray-400">
  <a href="https://open.spotify.com/artist/abc" target="_blank">Bicep</a>
  <span class="text-gray-600">,&nbsp;</span>
  <a href="https://open.spotify.com/artist/def" target="_blank">Four Tet</a>
</ScrollingText>
```

<Tip>
  The `className` prop is intentionally named `className` (not `class`) because `class` is a reserved word in JavaScript and cannot be destructured from `$props()` in Svelte 5 without aliasing. Use `className` when passing styles from the parent.
</Tip>

<Note>
  Overflow calculation is deferred by one microtask (`setTimeout(..., 0)`) to let the browser paint before measuring. This prevents incorrect measurements when `text` or `children` changes synchronously.
</Note>

## States

| Condition | Behaviour |
|---|---|
| Text fits container | `inline-block truncate w-full` â€” single line with ellipsis |
| Text overflows | Duplicate appended, edge fade applied, `animate-scroll-text` class added |
| `text` is empty string and no `children` | Container is empty; no animation runs |

## CSS animation

The `scrollText` keyframe is defined in the component's `<style>` block:

```css
@keyframes scrollText {
  0%   { transform: translateX(0); }
  100% { transform: translateX(calc(-1 * var(--text-width))); }
}
```

`--text-width` is injected as an inline CSS variable set to the measured pixel width of the scrolling content, ensuring the animation always moves exactly one full copy width regardless of text length.
