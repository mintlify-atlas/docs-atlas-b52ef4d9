The accent color system lets visitors pick a personal highlight color that persists across sessions. The server reads the color from a cookie and injects it as CSS variables into the HTML at render time, preventing any flash of the default color.

## Default color

The default accent color is `#e16666` — a muted coral red. It is used when no `accent_color` cookie is present.

## How the server hook works

The logic lives in `hooks.server.js` and runs on every request via SvelteKit's `handle` hook:

```js
export const handle = async ({ event, resolve }) => {
  const accentColor = event.cookies.get('accent_color') || '#e16666';

  const hexToRgbStr = (hex) => {
    if (!hex || hex.length < 7) return '225, 102, 102';
    const r = parseInt(hex.slice(1, 3), 16);
    const g = parseInt(hex.slice(3, 5), 16);
    const b = parseInt(hex.slice(5, 7), 16);
    return `${r}, ${g}, ${b}`;
  };

  const accentRgb = hexToRgbStr(accentColor);

  const response = await resolve(event, {
    transformPageChunk: ({ html }) =>
      html
        .replace('%accent%', accentColor)
        .replace('%accent_rgb%', accentRgb),
  });

  return response;
};
```

<Steps>
  <Step title="Read the cookie">
    `event.cookies.get('accent_color')` reads the stored hex value. If absent, it falls back to `'#e16666'`.
  </Step>

  <Step title="Convert hex to RGB components">
    `hexToRgbStr` parses the six-character hex string into three decimal integers separated by commas (e.g., `'225, 102, 102'`). This format is required for the `rgba()` CSS function.
  </Step>

  <Step title="Inject into the HTML">
    `transformPageChunk` performs two string replacements on the raw HTML before it is sent to the browser. The placeholders `%accent%` and `%accent_rgb%` must be present in `app.html`:

    ```html
    <!-- src/app.html -->
    <style>
      :root {
        --accent: %accent%;
        --accent-rgb: %accent_rgb%;
      }
    </style>
    ```

    Because this runs on the server before the page is streamed, the correct values are in place before the browser parses any CSS. There is no flash of unstyled content.
  </Step>
</Steps>

## CSS variables

After injection, two CSS custom properties are available globally:

| Variable | Example value | Usage |
|----------|---------------|-------|
| `--accent` | `#e16666` | Solid color fills, borders, text |
| `--accent-rgb` | `225, 102, 102` | Alpha-channel values via `rgba()` |

## Using accent color in components

<Tabs>
  <Tab title="Tailwind classes">
    The Tailwind config maps `accent` to `var(--accent)`, so you can use standard utility classes:

    ```svelte
    <!-- Solid text color -->
    <span class="text-accent">highlighted text</span>

    <!-- Solid background -->
    <div class="bg-accent">filled element</div>

    <!-- Border with opacity -->
    <div class="border border-accent/30">subtle border</div>

    <!-- Background with opacity -->
    <div class="bg-accent/10">tinted background</div>
    ```
  </Tab>

  <Tab title="Inline styles">
    When you need arbitrary opacity values or the color inside a CSS expression, reference `--accent-rgb` directly:

    ```svelte
    <!-- Semi-transparent overlay -->
    <div style="background: rgba(var(--accent-rgb), 0.15);">...</div>

    <!-- Box shadow -->
    <div style="box-shadow: 0 0 20px rgba(var(--accent-rgb), 0.4);">...</div>

    <!-- CSS gradient -->
    <div style="background: linear-gradient(135deg, rgba(var(--accent-rgb), 0.2), transparent);">...</div>
    ```
  </Tab>

  <Tab title="CSS">
    In `.css` or `<style>` blocks you can consume either variable:

    ```css
    .glow {
      color: var(--accent);
      text-shadow: 0 0 8px rgba(var(--accent-rgb), 0.6);
    }

    .card:hover {
      border-color: var(--accent);
      background: rgba(var(--accent-rgb), 0.05);
    }
    ```
  </Tab>
</Tabs>

## Client-side state

A global `themeState` Svelte rune in `src/lib/state.svelte.js` mirrors the active accent color on the client:

```js
export const themeState = $state({
  accentColor: null,
  accentRgb: null,
});
```

When a user picks a new color, the UI updates `themeState` and writes the new value to the `accent_color` cookie:

```js
function applyAccentColor(hex) {
  themeState.accentColor = hex;
  // Update CSS variable immediately for instant preview
  document.documentElement.style.setProperty('--accent', hex);

  // Persist to cookie for SSR on next request
  document.cookie = `accent_color=${hex}; max-age=31536000; path=/; samesite=lax`;
}
```

The `document.documentElement.style.setProperty` call makes the change visible instantly without a page reload. The cookie ensures the correct color is injected by the server on subsequent visits.

## FOUC prevention

Because `transformPageChunk` replaces `%accent%` and `%accent_rgb%` before the browser receives any bytes, the CSS variables are correct from the very first paint. There is no flash of the default coral red for users with a saved custom color.

<Tip>
  If you are adding a new component that uses the accent color, always reference `var(--accent)` or `var(--accent-rgb)` rather than hardcoding a color value. This ensures the component respects the user's saved preference automatically.
</Tip>

## How users change the accent color

The site exposes a color picker that calls `applyAccentColor(hex)` on selection. The change takes effect immediately via the CSS variable update, and is persisted in the `accent_color` cookie with a one-year expiry (`max-age=31536000`). On the next page load the server reads the cookie and injects the saved color, so the correct accent is present before any JavaScript runs.
